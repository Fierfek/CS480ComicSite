1. Introduction


        This document is a Software Requirement Specification (SRS) for ComicBash, a comic fan website to be designed by WebTech, LLC. This is the initial draft for the SRS and will be used for the extensions. This document is prepared as per IEEE conventions for software requirement specification.
        The purpose of this project is to provide a community driven website for fans of comic books that will combine ease of use with a rich selection of social options for the reporting and discussion of comic books


1.1 Definitions


        Henceforth in this document, the client Prof. Edwin Rodrìguez will be referred to as simply client, and Webtech, LLC will be referred to as provider.
        This SRS will be presented in three sections. Section 1 will be the introduction, and will serve to familiarize the reader with the basics of the project. Section 2 will be the description, and will go more in depth with the goals and methods of the project. Section 3 will be the requirements, and will present technical information regarding the proposed project.


1.2 Purpose


        The goal of this document is to provide a complete description of the development of ComicBash. It is a basis for agreement between the client and the provider for the product to be developed. Through this document the functionality, external interfaces, performance, attributes, and design constraints of the project will be developed. Therefore, intended reader groups for this SRS are the client and the provider.

1.3 Overview


        This document will firstly focus on describing the project in terms of prospective, functionality, user characteristics, assumptions, and dependencies. Then, the document will address specific requirements for the project, including external interface requirements, requirements for the website itself, performance requirements, and other requirements.

2. Overall Description

        This section provides a background about the specific requirements of the ComicBash website to be developed. Even though we will not describe every requirement in detail, this section will describe the elements that affect the final product.  

2.1 Product Perspective

        This website is intended for anyone who has an interest in comic books. The website will be the main interface where users can interact with other fellow comic book enthusiasts, and find their favorite comic books to. The main interface will allow users to navigate through the website such as searching for a specific comic book, seeing the latest updates on their favorite comic books, and finding out about the newest releases.

        In order for users to use the product, users are required to create an account. Once the account has been made it will be saved into the database, so the website will be able to remember the user he/she logs back on. In addition to that, each registered user will have their own profile which consist of their basic information, a profile picture, and a section where other users can leave comments on.  

2.2 Product Functions

        This product will have some features that permits users to interact with other users, and search for information on a diverse set of comic books. The required functionalities are divided in 3 categories: external requirements, system requirements, performance requirements and database requirements.

2.2.1 Community Requirements

This category requirements is related to users. Users can create an account to add their picture and their preferences for comic books, post comments about different comic books and follow comic books or/and other users.

 

2.2.2 ComicBash Requirements

        Each comic book on the ComicBash will consist of a title, it’s volume, issue, series, writer, illustrator,  characters, publisher, a synopsis of the plot, and a cover art. In addition to that, there will also be a overall rating based on the collection of the community’s scoring for each comic book. All users are able to view the overall rating plus they are able to give their own personal score to add on to the overall score. Besides from providing a synopsis of the plot, there will be a spoiler section where users are able to reveal on their own discretion to see the entire plot of the specific comic book.

2.2.3 Search Engine Requirements

        An important part of a comic website is the search engine. Users may search for specific a comic book, issue, developer, year, or another user.    

2.3 User Characteristics

        Users of this community driven website will be comics book fans who enjoy keeping up with news about comic books and issues of diverse kind of comic books.

2.4 Constraints

        Portability is the key factor in making this website by being accessible on all computers and mobile devices. In other words, the site needs be able to run all different kinds of web browsers.

        Another aspect to take into account is the handling of sensitive information from users. Security is important since all of the users’ information will be stored into the database. In addition to security, the website should not allow users to have access to other users information through the database.

  1. Email is alphanumeric
  2. Password field is alphanumeric and hidden
  1. Output: Display error message if login is unsuccessful.
  2. Process: Send input data to server for processing and receive a result on weather the user is in the database.
  1. Link to account creation page
  2. Forgot password
  1. User inputs email (alphanumeric) then hits submit to be redirected to security questions if email is valid
  1. On fail give error
  2. On success forward to security questions
  1. Lists the users question, along with 2 textboxes where the user can put their answer in the textbox.

  2. If the answers match forward the user to password reset page
  3. On fail give the proper error
  1. Password reset

  1. Username is alphanumeric and can not be previously used
  2. Full name accepts characters only
  3. Email must be in valid email format and cannot be previously used
  4. Password field is alphanumeric and should be hidden

3.4 Database Requirements

        CamicBash will use DynamoDB from AWS to store data. This is a non relational database.

3.4.1 Tables

  • Users - User data such as email and username
  • User Events - User generated events such as comments and ratings
